Overview

¿Qué es Claude Code?

Un agente de línea de comandos creado por Anthropic.
No es un chatbot — actúa, razona y ejecuta en tu codebase.

🧠

Agente Autónomo

Lee archivos, escribe código, ejecuta comandos y hace commits por ti.

🔧

Integración Total

Funciona con cualquier lenguaje, framework o herramienta en tu terminal.

🔌

Extensible

Conecta bases de datos, APIs y servicios via MCP. Automatiza con Hooks.

  • →
    Lee y edita archivos, busca en el codebase, ejecuta tests
  • →
    Hace commits, gestiona git, crea Pull Requests
  • →
    Entiende el proyecto completo a través de CLAUDE.md
  • →
    Se integra en pipelines CI/CD en modo headless

Process

Flujo de Trabajo

Describe → Claude lee → actúa → iteras.
Como un ingeniero senior en tu terminal.

💬

1. Describir

Tarea en lenguaje natural con contexto específico

🔍

2. Analizar

Lee archivos, entiende el contexto, formula un plan

⚙️

3. Actuar

Edita, ejecuta tests, hace commits

🔄

4. Iterar

Feedback en el mismo chat, ajusta sin perder contexto

# Ejemplo completo de sesión
claude "el endpoint POST /users falla con 500 cuando el email ya existe. Manéjalo con status 409"

# Claude: lee UserController.ts → identifica el bug
#         modifica el código → corre los tests
#         todo verde ✓

/review   # revisa el diff antes de commitear
"crea un commit semántico para estos cambios"

Context

CLAUDE.md

Claude lo lee automáticamente en cada sesión.
Es el contexto permanente de tu proyecto.

📂 Jerarquía

~/.claude/CLAUDE.md    # todos los proyectos
./CLAUDE.md            # proyecto raíz
./src/CLAUDE.md        # subdirectorio

💡 Qué incluir

  • → Stack y versiones
  • → Comandos de build y test
  • → Convenciones del equipo
  • → Qué NO modificar
CLAUDE.md
# API de Usuarios

## Stack
- Node.js 20 + TypeScript 5
- Express + Prisma + PostgreSQL
- Tests: Vitest + Supertest

## Comandos
npm run dev # desarrollo
npm test # tests
npm run build # compilar

## Convenciones
- Variables en camelCase
- Errores con clase AppError

## ⚠ NO modificar
# /src/generated/ (auto-generado)
# Tests deben pasar antes de commit

"rules": [...]→ JSON estructurado
📝

Estilo de código

Naming, formato, idioma de comentarios

🚫

Restricciones

Archivos intocables, patrones prohibidos

🔄

Workflow

Siempre testear, commitear semánticamente

🌐

Idioma

Responder en español, docs en inglés

CLAUDE.md — sección Rules
## Rules

# Estilo
- Usa camelCase para variables y funciones
- Usa PascalCase para clases e interfaces
- Comentarios en español, código en inglés

# Testing
- Nunca hagas commit sin tests pasando
- Mantén cobertura > 80% en src/

# Archivos protegidos
- NO modificar src/generated/
- NO tocar .env ni secrets/

# Git
- Commits en formato Conventional Commits
- Una tarea = una rama = un PR

⚙️ Rules en settings.json — control granular por herramienta

// ~/.claude/settings.json
{
  "rules": [
    "Responde siempre en español",
    "Usa TypeScript strict mode en todo código nuevo",
    "Nunca uses console.log — usa el logger de la app",
    "Antes de editar un archivo, léelo completo"
  ]
}
Automatización · Skills

SKILL.md

Comandos reutilizables que Claude ejecuta con /skill-name. Define el prompt una vez, invócalo siempre.

📂 Dónde colocarlos

~/.claude/skills/ ← global
commit.md→ /commit
review-pr.md→ /review-pr

.claude/skills/ ← proyecto
seed-db.md→ /seed-db
⚡

Invocación directa

Escribe /skill-name en cualquier sesión de Claude

🔁

Compartible

Versiona los .md junto al repo del proyecto

~/.claude/skills/commit.md
---
name: commit
description: Crea un commit semántico
---

# Crear Commit Semántico

Analiza los cambios en staging y crea
un commit siguiendo Conventional Commits:

1. Ejecuta git diff --cached
2. Determina tipo: feat/fix/chore/docs
3. Escribe mensaje conciso en inglés
4. Incluye Co-Authored-By si aplica

# Uso: /commit

⚡ Instalación e invocación

# Crear skill global
mkdir -p ~/.claude/skills && cat > ~/.claude/skills/commit.md << 'EOF'
---
name: commit
description: Crea un commit semántico siguiendo Conventional Commits
---
Analiza git diff --cached y genera el commit apropiado.
EOF

# Invocar dentro de una sesión de Claude Code
/commit       # ejecuta el skill "commit"
/review-pr    # ejecuta el skill "review-pr"
/seed-db      # skill local del proyecto

Multi-Agent · CLI

Subagentes

Claude Code lanza subagentes especializados en worktrees aislados. Cada subagente tiene su propio contexto, herramientas y permisos — sin interferir entre sí.

⚡
Claude Code CLI
orquestador principal
↓   lanza subagentes
🔍
Explore
📋
Plan
⚙️
Execute
⚡

Paralelo

Corren simultáneamente sin bloquearse.

🔒

Aislado

Git worktree propio por subagente.

🎯

Especializado

Cada uno con su rol y CLAUDE.md.

♾️

Sin límite

Tareas distribuidas en N agentes.

INTERACTIVO — sesión normal
# Claude lanza subagentes automáticamente
# al detectar tareas complejas o paralelas
claude

> "Refactoriza el módulo auth y en paralelo genera los tests unitarios"

# Claude Code internamente:
# → Subagente 1 (worktree-1): refactoriza auth/
# → Subagente 2 (worktree-2): genera tests/auth/
# → Orquestador: fusiona resultados y resuelve conflictos
HEADLESS — CI/CD sin intervención
# Modo no-interactivo para pipelines
claude --dangerously-skip-permissions \ -p "implementa el feature #247 con tests y PR"

# Flags útiles para subagentes en CI:
# --max-turns 50        limitar iteraciones
# --output-format json  parsear resultado
# --model claude-opus-4-5  modelo específico

# Cada subagente hereda permisos del orquestador
# y puede lanzar sus propios sub-subagentes
Automation

Hooks

Scripts que se ejecutan automáticamente
antes y después de cada acción de Claude.

PreToolUse

Antes de actuar

Ejecuta antes de Bash, Write, Edit. Un exit code ≠ 0 bloquea la acción por completo.

Bash Write Edit WebFetch
bloquear comandos peligrosos
// ~/.claude/settings.json
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "hooks": [{ "type": "command",
        "command": "bash ~/.claude/guard.sh"
      }]
    }]
  }
}

// guard.sh — exit 1 bloquea la acción
if echo "$CLAUDE_TOOL_INPUT" | grep -qE \
  "rm -rf|drop table|push --force"; then
  echo "Comando bloqueado por política"; exit 1
fi
PostToolUse

Después de actuar

Ejecuta después de cada herramienta. Ideal para formateo, tests, logs y notificaciones segun sea el caso necesario.

Prettier ESLint Tests Slack
formatear al guardar
// ~/.claude/settings.json
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Write|Edit",
      "hooks": [{ "type": "command",
        "command": "npx prettier --write \"$CLAUDE_FILE_PATH\""
      }]
    }],
    "Stop": [{
      "hooks": [{ "type": "command",
        "command": "osascript -e 'display notification \"Claude terminó\"'"
      }]
    }]
  }
}
Security

Sistema de Permisos

Tú controlas qué puede y qué no puede hacer Claude.
Configurable por proyecto o globalmente.

✓ Permitido por defecto

  • Leer archivos del proyecto
  • Buscar en el codebase
  • Ejecutar git status / diff
  • Crear archivos nuevos
  • Correr tests

⚠ Requiere confirmación

  • Modificar archivos existentes
  • git commit / git push
  • Instalar dependencias
  • Eliminar archivos
  • Acceso a red externa
// ~/.claude/settings.json — lista allow / deny
{
  "permissions": {
    "allow": ["Bash(npm test)", "Bash(npm run lint)", "Write(src/**)"],
    "deny":  ["Bash(rm -rf*)", "Bash(git push --force)"]
  }
}
Best Practices

Mejores Prácticas

Lo más importante de la certificación de Anthropic.

  • 1
    Contexto específico — menciona archivo, función y comportamiento esperado. Evita "arregla los bugs".
  • 2
    CLAUDE.md actualizado — stack, comandos, convenciones y qué NO modificar.
  • 3
    Tareas atómicas — divide tareas grandes. Es más fácil verificar cambios pequeños.
  • 4
    Usa /compact en sesiones largas para mantener eficiencia y reducir costos.
  • 5
    Revisa con /review antes de confirmar. No apruebes ediciones masivas sin revisar.
  • 6
    Comandos personalizados — crea /deploy, /changelog, /security para flujos repetitivos del equipo.
  • 7
    Hooks de calidad — formatea y valida automáticamente con PostToolUse.
  • 8
    Headless en CI/CD — integra con claude -p "..." --output json en pipelines.
